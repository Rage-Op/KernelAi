<objective>
Land the Playwright headful browser tool (HANDS-03): a `tools/browser.ts` adapter that opens ONE headful Chromium via `chromium.launchPersistentContext` against a DEDICATED profile dir (never the user's real Chrome), and exposes navigate / scrape / fill / login. Every navigation logs the full URL + provenance; scraped content is tagged `source:'external'`; the `fill` op runs the SAME credential fence as the Peekaboo `type` op (02-01) — refusing labelled-password / sensitive-autocomplete fields BEFORE typing. Per the unit-vs-manual split (RESEARCH.md Validation Architecture): navigate/scrape/fill against a local `file://` fixture page, the dedicated-profile-dir assertion, full-URL+provenance logging, and the fence are ALL UNIT-tested (no live network). FORBIDDEN (RESEARCH.md / STACK.md "What NOT to Use"): pointing `userDataDir` at the real Chrome profile (broken by Chrome policy + risks the owner's real cookies); headless + stealth plugins (brittle, invites login-grinding). Dedicated dir, headful, no stealth.

The only npm install here is `playwright@1.61.0`, pinned EXACTLY (no caret), plus `npx playwright install chromium` (the browser binary — a setup step, gitignored-by-location).

Purpose: KERNEL gains a browser hand behind the single gate, with egress logging and the credential fence wired in.
