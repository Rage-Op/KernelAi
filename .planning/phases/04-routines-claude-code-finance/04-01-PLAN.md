<objective>
Build the morning-brief routine engine and the four Phase-4 Face widgets it choreographs. The engine loads `routines/morning-brief.yaml` (presets Workday/Weekend/Travel; each step `enabled`/`order`/`params`/`tier`), runs the enabled steps in `order` serially, and per narrated step emits ONE `speak` frame via the shipped `assembleSpeak` so the Stage blooms one-or-two widgets — never a static grid. mail_triage tags messages with the always-on 7B `helper.classify`. Widget data-binding reuses the shipped `widget.data` → `AppCoordinator.presentedData` path (no new IPC arm here).
