<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Plaid API → daemon | Balance/transaction payloads (incl. memos/payees) are `source:external` untrusted data |
| Bank OAuth flow → KERNEL | The bank authenticates; KERNEL receives a read-only token only — never types creds |
| Finance store → kernel-memory git repo | The encrypted DB must NEVER cross into git history (the existential leak) |
| DB key → macOS Keychain | The key lives only in the Keychain; never on disk / in the memory repo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Information Disclosure | finance data committed to git | mitigate | 4-layer stack: gitignore (a) + pre-push byte scan (b, deliberate-abort proven) + at-rest encryption (c) + startup ls-files assertion (d), all in the kernel-memory repo |
| T-04-07 | Cryptography failure | finance.db at rest | mitigate | SQLCipher AES-256 (cipher_compatibility=4); wrong-key open fails; no hand-rolled crypto |
| T-04-08 | Information Disclosure | DB key leakage | mitigate | key in Keychain via `security` CLI only; never written to a file or the memory repo; never logged |
| T-04-09 | Spoofing / Info Disclosure | bank credential entry | mitigate | finance is read-only OAuth; no type/fill op exists in the finance tool; the shipped credential fence still backstops any peekaboo/browser path |
| T-04-10 | Tampering / EoP | Plaid memo as instruction | mitigate | Plaid content is external data, never promoted to knowledge/IDENTITY; not fed as instruction |
| T-04-11 | Errors & Logging | logging finance values/key/token | mitigate | pino logs sync events only — never amounts, the DB key, or the access_token (ASVS V7) |
| T-04-SC | Tampering | npm installs (plaid, better-sqlite3-multiple-ciphers) | mitigate | both slopcheck [OK]/Approved in 04-RESEARCH audit; exact pins; native build is the standard better-sqlite3 prebuild mechanism (no [ASSUMED]/[SUS] → no blocking checkpoint) |
