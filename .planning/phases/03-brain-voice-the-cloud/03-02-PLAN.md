<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Face -> daemon | Mic PCM / WAV path (untrusted audio payload) crosses into the daemon |
| daemon -> whisper-cli subprocess | The daemon spawns an external binary; spawn args must not be shell-interpolated |
| whisper-cli stdout -> daemon | Transcript text is external-sourced content (carries `source:'external'` provenance downstream) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | DoS | whisper-cli absent (spawn ENOENT) crashing the loop | mitigate | Probe-then-escalate (mirrors peekaboo.ts): catch ENOENT/non-zero exit, return a typed escalation, never throw across the loop boundary. Proven by whisper.test.ts case (b). |
| T-03-07 | Injection | spawn arg / audio path handling | mitigate | spawn with an explicit argv array (NO shell string); audio path passed as a discrete arg; no interpolation. |
| T-03-08 | Tampering/EoP | transcript treated as instruction | mitigate | Transcript is external-sourced content — it enters as an utterance and any resulting tool action still passes gate.authorize (Phase-2 lock). This wrapper does not bypass the chokepoint. |
</threat_model>
